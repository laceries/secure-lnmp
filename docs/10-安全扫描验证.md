# 10 - 安全扫描验证

## 目标

使用 nmap 和 Nikto 对公网 Web 站点进行安全扫描，验证端口暴露情况和 Web 安全加固效果，并根据扫描结果进行整改。

## 背景

安全加固不能只停留在“配置完成”，还需要通过工具验证实际效果。本项目使用：

| 工具 | 用途 |
|---|---|
| nmap | 检查公网端口暴露和服务识别 |
| Nikto | 检查 Web 常见安全配置问题 |

扫描结果用于验证：

- 3306 / 33060 是否未暴露公网；
- 是否仅开放必要端口；
- Nginx / PHP / WordPress 指纹是否隐藏；
- WordPress 敏感路径是否被拦截；
- 安全响应头是否生效。

## nmap 关键端口扫描

扫描命令：

```bash
nmap -Pn -sV -p 22,80,443,3306,33060 example.com
```

参数说明：

| 参数 | 含义 |
|---|---|
| `-Pn` | 不进行 ping 探测，直接扫描端口 |
| `-sV` | 尝试识别服务版本 |
| `-p` | 指定扫描端口 |

期望结果：

| 端口 | 期望状态 | 说明 |
|---|---|---|
| 22 | open | SSH |
| 80 | open | HTTP，用于跳转 HTTPS 和证书续期 |
| 443 | open | HTTPS 正式服务 |
| 3306 | closed / filtered | MySQL 不应公网开放 |
| 33060 | closed / filtered | MySQL X Protocol 已关闭 |

## nmap Top 100 端口扫描

```bash
nmap -Pn --top-ports 100 example.com
```

目标：

- 验证除 22 / 80 / 443 外，没有暴露常见高风险端口；
- 避免 Redis、MongoDB、Elasticsearch、MySQL 等服务意外暴露。

期望仅看到：

```text
22/tcp open
80/tcp open
443/tcp open
```

## Nikto 扫描

扫描命令：

```bash
nikto -h https://example.com -ssl -Format txt -output nikto-example.txt
```

Nikto 初次扫描发现的主要问题包括：

```text
X-Powered-By: PHP/8.2.10
X-Redirect-By: WordPress
ETag 信息泄露提示
```

这些属于版本/指纹信息暴露，不一定是高危漏洞，但在安全加固项目中应尽量修复。

## 扫描后整改

### 1. 隐藏 PHP 版本

PHP 配置：

```ini
expose_php = Off
```

Nginx 双保险：

```nginx
fastcgi_hide_header X-Powered-By;
```

### 2. 隐藏 WordPress 指纹响应头

```nginx
fastcgi_hide_header X-Redirect-By;
```

### 3. 关闭 ETag

```nginx
etag off;
```

### 4. 保留安全响应头

```nginx
add_header X-Content-Type-Options "nosniff" always;
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
```

## 敏感路径验证

执行：

```bash
for URL_PATH in /xmlrpc.php /readme.html /license.txt /wp-config.php; do
  echo "===== $URL_PATH ====="
  curl -I "https://example.com$URL_PATH" | head -8
done
```

期望：

| 路径 | 期望状态 |
|---|---|
| `/xmlrpc.php` | 403 |
| `/readme.html` | 403 |
| `/license.txt` | 403 |
| `/wp-config.php` | 403 或 404 |

## 复扫结果

整改后 Nikto 输出中：

- 不再出现 `X-Powered-By: PHP/8.2.10`；
- 不再出现 `X-Redirect-By: WordPress`；
- 不再出现 ETag 信息泄露提示；
- 安全响应头仍然保留。

剩余提示中，Nikto 可能会显示：

```text
Uncommon header 'x-content-type-options'
Uncommon header 'x-frame-options'
Uncommon header 'referrer-policy'
Uncommon header 'x-xss-protection'
```

这些是主动添加的安全响应头，不是漏洞。

## 验证命令摘要

```bash
curl -I https://example.com/ | grep -Ei "x-powered-by|x-redirect-by|etag" || echo "敏感响应头未发现"
```

期望：

```text
敏感响应头未发现
```

## 安全意义

通过扫描验证，本项目实现：

- 数据库端口不暴露公网；
- Web 服务仅开放必要端口；
- Nginx、PHP、WordPress 指纹减少；
- WordPress 敏感路径被拦截；
- 安全响应头生效；
- 扫描发现问题能够闭环整改。

## 面试可讲点

可以这样讲：

> 我使用 nmap 和 Nikto 对站点做了扫描验证。nmap 验证公网仅开放 22、80、443，MySQL 3306 和 33060 未暴露；Nikto 初扫发现 PHP 版本、WordPress 响应头、ETag 等指纹泄露问题，我通过修改 php.ini 和 Nginx fastcgi_hide_header、etag off 等配置完成整改，并复扫确认问题消失。

## 注意事项

- 不要将包含真实 IP、真实域名、邮箱或路径敏感信息的原始扫描报告直接上传 GitHub；
- 上传报告前应脱敏；
- 扫描工具结果需要人工判断，不是所有提示都是漏洞；
- 安全扫描应形成“发现 → 分析 → 修复 → 复测”的闭环。
