# 01 - 环境准备

## 目标

将全新 Ubuntu 24.04 服务器准备好，安装编译 LNMP 所需的基础工具。

## 操作环境

| 项目 | 值 |
|------|------|
| 系统 | Ubuntu 24.04 LTS |
| 用户 | root |
| 时间 | 2026.04.28 |

## 操作步骤

### 步骤 1：更新系统

```bash
apt update && apt upgrade -y
```

* `apt update`：同步软件源列表
* `apt upgrade -y`：升级所有已安装的包到最新版本
* 为什么：确保系统处于最新状态，已知安全漏洞被修补

### 步骤 2：安装编译依赖

```bash
apt install -y build-essential libpcre2-dev libssl-dev zlib1g-dev wget
```

| 包名              | 作用                      |
| --------------- | ----------------------- |
| build-essential | GCC 编译器 + make 等基础编译工具链 |
| libpcre2-dev    | 正则表达式库（Nginx URL 匹配依赖）  |
| libssl-dev      | OpenSSL 开发库（HTTPS 依赖）   |
| zlib1g-dev      | 压缩库（gzip 依赖）            |
| wget            | 命令行下载工具                 |

## 验证

```bash
gcc --version
```

看到版本号（如 gcc 13.3.0）说明编译工具链就绪。

## 遇到的坑

* 更新后终端提示 outdated binaries，重新登录即可，不影响后续操作。
