## 五、改进建议

### 🔴 建议修复

1. **X-Pingback 泄露 xmlrpc.php** — `/secure-lnmp/` 页面返回响应头：
   ```
   X-Pingback: https://zhyru.top/xmlrpc.php
   ```
   WordPress 输出此头，暴露了 xmlrpc.php 路径。建议在 Nginx 中去掉：
   ```nginx
   # 建议加到 nginx.conf 的 server/location 块中
   proxy_hide_header X-Pingback;
   fastcgi_hide_header X-Pingback;
   ```
   或者从 WordPress 侧移除：
   ```php
   // functions.php
   remove_action('wp_head', 'rsd_link');
   ```

2. **WordPress REST API 指纹** — 所有页面返回：
   ```
   Link: <https://zhyru.top/wp-json/>; rel="https://api.w.org/"
   ```
   直接标识站点为 WordPress。如需去掉：
   ```php
   // functions.php
   remove_action('wp_head', 'rest_output_link_wp_head');
   remove_action('template_redirect', 'rest_output_link_header', 11);
   ```

### 🟡 建议关注

3. **磁盘使用 35%** — 当前安全，但如果启用了自动备份（Phase 9），需预留空间。
4. **PHP-FPM workers 3 个** — 当前负载低够用，如果后续加监控 exporter 需要留意端口和进程数。
5. **缺少监控和告警**（Phase 7 待建）— 没有对磁盘、内存、证书过期、服务存活的主动告警，目前只能人工检查。
